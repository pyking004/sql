system defined Data integrity:
=========================
I. Entity Integrity
	1. Primary key
	2. unique
II.Domain Integrity
	3. check
	4. not null
III.Referential Integrity
	5. foreign key


syntax:
=======
col1_name datatype(size) constraint <constraint_name> <constraint_type>
		(or)
col1_name datatype(size) <constraint_type>

PRIMARY KEY
===========
	It doesn't accept any duplicate values and null values.

Example1
========
drop table student1 purge;

create table student1
(
sid number(4) constraint student1_sid_pk primary key,
sname varchar2(15),
course varchar2(10),
fee number(9,2)
);

sql>desc student1

Note:describe table will not display any constraints which are created.

SQL>insert into student1 values(101,'scott','oracle',5000);

SQL>insert into student1 values(101,'smith','unix',4000);

ERROR at line 1:
ORA-00001: unique constraint (SCOTT.STUDENT_SID_PK) violated

SQL>insert into student1 values(102,'smith','unix',4000);

SQL>insert into student1 values(null,'scott','oracle',5000);

ERROR at line 1:
ORA-01400: cannot insert NULL into ("SCOTT"."STUDENT"."SID")

Note: One table will have only one primary key on any column.

Notes: 
======
*only one primary key for one table on any column.
*when primary key is created internally one default index will be created on that column.
*constraint names should be unique in the entire schema.

UNIQUE:
=======
*It doesn't accept any duplicate values
*Any number of unique constraints can be created on single table unlike primary key.
*Unique constraint doesn't check null values.
*so it accepts any number of null values.

drop table student4;

create table student4
(
sid number constraint student4_sid_unq unique,
sname varchar2(15),
course varchar2(10),
fee number(7,2)
);

insert into student4 values(101,'scott','oracle',5000);
insert into student4 values(102,'smith','oracle',5000);
insert into student4 values(101,'scott','java',4500);

ERROR at line 1:
ORA-00001: unique constraint (SCOTT.STUDENT4_SID_UNQ) violated

insert into student4 values(null,'scott','java',4500);
insert into student4 values(null,'scott','java',4500);


CHECK: Constraint
=================
* It is allowing you to provide your own constraint condition value.
* It accepts only those values which are satifying the "CHECK" constraint condition value.

drop table student5;

create table student3
(
sid number,
sname varchar2(15),
course varchar2(10) constraint student3_course_chk check(course in ('python','react','java')),
fee number(7,2)
);


insert into student5 values(101,'scott','oracle',2500);
insert into student5 values(102,'smith','unix',1500);
insert into student5 values(103,'scott','linux',2000);

ERROR at line 1:
ORA-02290: check constraint (SCOTT.STUDENT5_COURSE_CHK) violated

Foreign key:
============
It is used to establish relationship among database tables.


customer
========
accno	n(5)=> accno>=1001,no null,no duplicate=> primary key,check
acname	v2(25) ==> no nulls  => not null
actype	char   ==> no nulls, should be in(s,c,j) ==> not null, check
bal	n(9,2) ==> no nulls, bal>=5000  ==>  not null , check

transaction
============
accno 	n(5)  ==> valid account number => foreign key
name	v2(25) ==> not null            => not null
amount	n(9,2) ==> amount>=100,no null => check , not null
t_id	n(8)   ==> t_id >=500001,no nulls , no duplicate  => check , not null , unique
 

drop table customer purge;

create table customer
(
accno number(5) 
constraint cust_accno_pk primary key 
constraint cust_accno_chk check(accno>=1001),
acname varchar2(25)
constraint cust_acname_nn not null,
actype char
constraint cust_actype_chk check(actype in('s','c','j')) 
not null,
bal number(9,2) 
constraint cust_bal_chk check(bal>=5000) 
not null
);


drop table transaction purge;

create table transaction 
(
accno number(4) constraint trn_accno_fk references customer(accno),
name varchar2(15) not null,
amount number(9,2) not null,
tid number(8) constraint trn_tid_chk check(tid>=500001)
constraint trn_tid_unique unique
);


UNIQUE + NOT NULL != PRIMARY KEY(INDEX)

Notes:
======
* we create foreign key on those table which are having some relationship.
* In which table we are creating foreign key that is known as child table.
* To create a foreign key we required referenced table column.
* The referenced table is known as parent table so relationship is called parent child relationship.
* To establish foreign key successfully referenced table column should have primary key or unique  constraint.
* Once foreign key is established succeesfully then droping parent table is not possible.
* if you want to drop parent table first remove foreignkey or drop child table.
* deleting parent records not possible when you found any dependent child records in child table.
* one child table always depends on only one parent table column

constraint types:
=================

user_cons_columns
==================
	It is system defined table which maintains constraints columns information

DROPING THE CONSTRAINTS:
========================

syntax:
=======
	alter table <tablename> drop constraint <cons name>;


Droping primary key:
====================

sql>alter table customer drop constraint cust_accno_pk;

Note: primary key can not be dropped if at all any reference key is found.

* first remove foreign key then primary key

droping foreign key:
====================
sql>alter table transaction drop constraint trn_accno_fk;

Now remove primarykey
=====================
sql>alter table customer drop constraint cust_accno_pk;


droping unique constraint:
==========================
sql>alter table transaction drop constraint trn_tid_unq;


Note: 
=====
*constraints can not be modified 
*constraints can be added or dropped
*only one constraint which can be modified that is NOT NULL

On delete cascade:
==================
	it is used to delete the parent records even there are child dependent records in child table.

action on child:
================
if you delete parent records all dependent child records will be deleted.


Droping foreign key:
====================

alter table transaction drop constraint trn_accno_fk;

adding foreign key constraint with "On delete cascade"
======================================================

alter table transaction add constraint trn_accno_fk foreign key(accno) references customer(accno) on delete cascade;

delete from customer where accno=10001;

select * from customer;

select * from transaction;

On delete set null:
===================
	it is used to delete the parent records even there are child dependent records in child table.

Action On child:
================
Instead of deleting child records it will fill the null values in foreign key column values.

Again drop foreign key and create with "on delete set null" option
==================================================================
alter table transaction drop constraint trn_accno_fk;

alter table transaction add constraint trn_accno_fk foreign key(accno) references customer(accno) on delete set null;

select * from customer;

select * from transaction;

delete from customer where accno=10002;

select * from customer;

select * from transaction;
