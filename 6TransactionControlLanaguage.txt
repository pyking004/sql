Transaction Control Language:
=============================

COMMIT , ROLLBACK , SAVEPOINT

Request  => select * from emp;

Response  => empdata in parcel format with security

client software => sql*plus, sqlplus, sql developer, toad 

server software => rdbms from oracle

transaction => request and response

buffer  =>  Tempararoy memoray loaction to process t.transactions

commit: making t.t as p.t so that database will be effected and buffer clears

rollback: cancelling all t.t from buffer and buffer will be cleared

savepoint : It is used to control the rollbacking.

Transaction control language:
=============================
Example1:
=========
sql>drop table t purge;
sql>create table t(c1 number(3));
sql>insert into t values(100);
sql>select * from t;
100
sql>rollback;
rollback completed...
sql>select * from t;

Example with savepoint:
======================
drop table t purge;
create table t(c1 number(3));
insert into t values(100);
savepoint s1;
insert into t values(200);
savepoint s2;

savepoint s3;
insert into t values(400);

Creating savepoint:
==================
syntax:
=======
	savepoint <savepointname>;
example:
========
sql>savepoint s1;

Notes:
====== 
*When you apply commit or rollback all savepoints which are  established will be cleared from Buffer.
*Rollback to perticular savepoint will not be cleared buffer
